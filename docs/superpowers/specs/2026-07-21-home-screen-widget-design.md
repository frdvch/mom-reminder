# Домашній екран-віджет "Найближче нагадування"

## Контекст

Застосунок "Мама не забудь…" зараз вимагає відкрити застосунок, щоб побачити
список нагадувань. Для основного юзера (мама, 51 рік, не дуже технічна людина)
це зайвий крок — простіше побачити найближче нагадування прямо на робочому
столі телефону і, за потреби, одразу позначити його виконаним, не заходячи в
застосунок.

## Що показує віджет

Тільки **одне найближче активне нагадування** (не список) — назва + час.
Мінімальний розмір комірки на екрані, найлегше читати. Якщо активних
нагадувань немає — "Немає активних нагадувань", кнопка прихована.

## Компоненти

- **`widget/ReminderWidgetProvider.kt`** (новий пакет) — `AppWidgetProvider`.
  `onUpdate()` читає найближче активне нагадування і рендерить `RemoteViews`
  (Compose не працює у widgets — стандартний XML-layout).
- **`res/layout/reminder_widget.xml`** — назва, час, кнопка "Виконано".
- **`res/xml/reminder_widget_info.xml`** — `AppWidgetProviderInfo`: розмір ~2x1
  (мінімальний, resizable), `updatePeriodMillis` ~30 хв (мінімум, який дозволяє
  Android) як страховка на випадок пропущеного форс-рефрешу.
- **`ReminderDao.getNextActive(): Reminder?`** — нова query
  (`SELECT * FROM reminders WHERE isActive = 1 ORDER BY dateTimeEpochMillis ASC LIMIT 1`),
  тривіальне розширення наявного `ReminderDao.kt`.

## Оновлення даних

Віджет **не** покладається на періодичний таймер як основний механізм — він
оновлюється відразу після зміни даних:

- `ReminderRepository.save()` / `.delete()` / `.advanceToNextOccurrence()` після
  виконання шлють широкомовний `APPWIDGET_UPDATE` intent для власного provider'а
  (стандартний Android-патерн форс-рефрешу віджета, без нових залежностей).
- `BootReceiver.catchUpMissedAndRescheduleActive()` — так само форсить оновлення
  після catch-up, бо часи нагадувань могли змінитись.

## Дія "Виконано" з віджета

Кнопка на віджеті шле кастомний broadcast (окрема `action`, не системний
`APPWIDGET_UPDATE`) у той самий `ReminderWidgetProvider`. `onReceive()`
розпізнає цю дію, робить `goAsync()` (як `BootReceiver`), викликає вже наявний
**`ReminderCompletion.markDone(repo, reminder)`** — той самий код, що й
повноекранний алярм використовує для "Виконано" — і після завершення форсить
оновлення віджета.

Тап по решті поверхні віджета (не по кнопці) відкриває `MainActivity`.

## Помилки/edge-кейси

- Немає активних нагадувань → плейсхолдер-текст, кнопка прихована.
- Нагадування видалено/позначено виконаним іншим шляхом (наприклад, з форми в
  застосунку) саме в момент рендеру віджета — обробляється тим самим форс-
  рефрешем після `save`/`delete`, окремої синхронізації не потрібно.

## Верифікація

- Компіляція + `gradle assembleDebug` — можна прогнати з сесії (як і для
  update-check фічі), без Android Studio GUI.
- Реальна поведінка віджета (додавання на робочий стіл, оновлення при зміні
  даних, клік "Виконано") — потребує реального пристрою чи емулятора з
  launcher'ом; це піде в той самий пункт "не перевірено на пристрої", що й
  update-check flow.
